## show_toast

Displays a toast notification to the user.

### Syntax

<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "message": "Operation completed!",
  "level": "success"
}
```
</CodeBlock>

### Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `message` | string | Yes | Notification message |
| `level` | string | - | Toast level (default: `info`) |
| `title` | string | - | Toast title |
| `duration` | number | - | Display duration (ms) |
| `action` | object | - | Action button |

### Levels

| Level | Color | Use Case |
|-------|-------|----------|
| `info` | Blue | General information |
| `success` | Green | Success confirmations |
| `warning` | Yellow | Warnings |
| `error` | Red | Error messages |

### Examples

**Basic toast:**
<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "message": "Settings saved"
}
```
</CodeBlock>

**Success toast:**
<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "message": "Item created successfully!",
  "level": "success"
}
```
</CodeBlock>

**Error toast:**
<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "message": "Failed to save changes",
  "level": "error"
}
```
</CodeBlock>

**Warning toast:**
<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "message": "Your session will expire in 5 minutes",
  "level": "warning"
}
```
</CodeBlock>

**With title:**
<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "title": "Success",
  "message": "Your profile has been updated",
  "level": "success"
}
```
</CodeBlock>

**With duration:**
<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "message": "Copied to clipboard",
  "level": "success",
  "duration": 2000
}
```
</CodeBlock>

**With dynamic message:**
<CodeBlock lang="json">
```json
{
  "type": "show_toast",
  "message": "Welcome back, {{state.user.name}}!",
  "level": "success"
}
```
</CodeBlock>

**From error response:**
<CodeBlock lang="json">
```json
{
  "type": "call_api",
  "api": "saveData",
  "on_error": [
    {
      "type": "show_toast",
      "message": "Error: {{$error.message}}",
      "level": "error"
    }
  ]
}
```
</CodeBlock>

### Common Patterns

**CRUD Operations:**

<CodeBlock lang="json">
```json
{ "type": "show_toast", "message": "Item created", "level": "success" }
{ "type": "show_toast", "message": "Changes saved", "level": "success" }
{ "type": "show_toast", "message": "Item deleted", "level": "success" }
{ "type": "show_toast", "message": "Operation failed", "level": "error" }
```
</CodeBlock>

**Copy to clipboard:**

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "icon": "Copy",
  "events": {
    "on_click": [
      { "type": "copy", "text": "{{state.shareUrl}}" },
      { "type": "show_toast", "message": "Link copied!", "level": "success", "duration": 2000 }
    ]
  }
}
```
</CodeBlock>

**After navigation:**

<CodeBlock lang="json">
```json
{
  "on_mount": [
    {
      "type": "conditional",
      "condition": "{{params.created}}",
      "then": [
        { "type": "show_toast", "message": "Item created successfully!", "level": "success" }
      ]
    }
  ]
}
```
</CodeBlock>

### Best Practices

1. **Be concise** - Keep messages short and clear
2. **Use appropriate levels** - Match level to message importance
3. **Confirm actions** - Show success after operations
4. **Include details in errors** - Help users understand what went wrong
5. **Don't overuse** - Avoid toast fatigue
