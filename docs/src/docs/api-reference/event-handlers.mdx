## Event Handlers

Events connect user interactions to actions. This reference covers all available events.

## Event Syntax

Events are defined in the `events` property:

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "text": "Click me",
  "events": {
    "on_click": [
      { "type": "show_toast", "message": "Clicked!" }
    ]
  }
}
```
</CodeBlock>

## Mouse Events

### on_click

Triggered when element is clicked.

<CodeBlock lang="json">
```json
{
  "events": {
    "on_click": [
      { "type": "update_state", "path": "clicked", "value": true }
    ]
  }
}
```
</CodeBlock>

**Context variables:**

- `$event` - Mouse event object

### on_double_click

Triggered on double-click.

<CodeBlock lang="json">
```json
{
  "events": {
    "on_double_click": [
      { "type": "navigate", "route": "/edit/{{state.item.id}}" }
    ]
  }
}
```
</CodeBlock>

### on_mouse_enter / on_mouse_leave

Triggered when mouse enters/leaves element.

<CodeBlock lang="json">
```json
{
  "events": {
    "on_mouse_enter": [
      { "type": "update_state", "path": "hovering", "value": true }
    ],
    "on_mouse_leave": [
      { "type": "update_state", "path": "hovering", "value": false }
    ]
  }
}
```
</CodeBlock>

## Form Events

### on_change

Triggered when input value changes.

<CodeBlock lang="json">
```json
{
  "type": "Field",
  "fieldType": "text",
  "name": "search",
  "events": {
    "on_change": [
      { "type": "update_state", "path": "searchQuery", "value": "{{$value}}" }
    ]
  }
}
```
</CodeBlock>

**Context variables:**

- `$value` - New input value
- `$event` - Change event object

### on_blur

Triggered when input loses focus.

<CodeBlock lang="json">
```json
{
  "events": {
    "on_blur": [
      { "type": "validateForm", "formId": "myForm" }
    ]
  }
}
```
</CodeBlock>

### on_focus

Triggered when input receives focus.

<CodeBlock lang="json">
```json
{
  "events": {
    "on_focus": [
      { "type": "update_state", "path": "inputFocused", "value": true }
    ]
  }
}
```
</CodeBlock>

### on_submit

Triggered when form is submitted.

<CodeBlock lang="json">
```json
{
  "type": "Form",
  "id": "contactForm",
  "events": {
    "on_submit": [
      { "type": "validateForm", "formId": "contactForm" },
      { "type": "call_api", "api": "submitContact" }
    ]
  }
}
```
</CodeBlock>

**Context variables:**

- `$event` - Submit event object
- Form data available via `form.formId`

## Keyboard Events

### on_key_down

Triggered when key is pressed.

<CodeBlock lang="json">
```json
{
  "events": {
    "on_key_down": [
      {
        "type": "conditional",
        "condition": "{{$event.key === 'Enter'}}",
        "then": [
          { "type": "call_api", "api": "submit" }
        ]
      }
    ]
  }
}
```
</CodeBlock>

**Context variables:**

- `$event.key` - Key pressed
- `$event.ctrlKey` - Ctrl modifier
- `$event.shiftKey` - Shift modifier
- `$event.altKey` - Alt modifier
- `$event.metaKey` - Meta modifier

### on_key_up

Triggered when key is released.

<CodeBlock lang="json">
```json
{
  "events": {
    "on_key_up": [
      { "type": "update_state", "path": "lastKey", "value": "{{$event.key}}" }
    ]
  }
}
```
</CodeBlock>

## Lifecycle Events

### on_mount

Triggered when page/component mounts.

<CodeBlock lang="json">
```json
{
  "on_mount": [
    { "type": "call_api", "api": "fetchInitialData", "storeAs": "data" }
  ]
}
```
</CodeBlock>

Common uses:

- Fetch initial data
- Initialize state
- Set up subscriptions

### on_unmount

Triggered when page/component unmounts.

<CodeBlock lang="json">
```json
{
  "on_unmount": [
    { "type": "update_state", "path": "cleanup", "value": true }
  ]
}
```
</CodeBlock>

Common uses:

- Clean up resources
- Save state
- Cancel pending operations

## Selection Events

### on_select

Triggered when selection changes.

<CodeBlock lang="json">
```json
{
  "type": "Table",
  "events": {
    "on_select": [
      { "type": "update_state", "path": "selectedRows", "value": "{{$value}}" }
    ]
  }
}
```
</CodeBlock>

**Context variables:**

- `$value` - Selected items

## Custom Events

### onEvent:[name]

Listen for custom events.

<CodeBlock lang="json">
```json
{
  "events": {
    "onEvent:itemSelected": [
      { "type": "update_state", "path": "selected", "value": "{{$event.data}}" }
    ]
  }
}
```
</CodeBlock>

Emit custom events:

<CodeBlock lang="json">
```json
{
  "type": "emit",
  "event": "itemSelected",
  "data": "{{state.currentItem}}"
}
```
</CodeBlock>

## Table/List Events

### on_row_click

Triggered when table row is clicked.

<CodeBlock lang="json">
```json
{
  "type": "Table",
  "events": {
    "on_row_click": [
      { "type": "update_state", "path": "selectedItem", "value": "{{$item}}" }
    ]
  }

```
</CodeBlock>

**Context variables:**

- `$item` - Row data
- `$index` - Row index

### onSort

Triggered when table sort changes.

<CodeBlock lang="json">
```json
{
  "type": "Table",
  "events": {
    "onSort": [
      { "type": "update_state", "path": "sortBy", "value": "{{$value.column}}" },
      { "type": "update_state", "path": "sortOrder", "value": "{{$value.order}}" }
    ]
  }
}
```
</CodeBlock>

### on_page_change

Triggered when pagination changes.

<CodeBlock lang="json">
```json
{
  "type": "Table",
  "events": {
    "on_page_change": [
      { "type": "update_state", "path": "page", "value": "{{$value}}" },
      { "type": "call_api", "api": "fetchPage" }
    ]
  }
}
```
</CodeBlock>

## Event Context Summary

| Variable | Type | Available In | Description |
|----------|------|--------------|-------------|
| `$value` | any | on_change, on_select | Current value |
| `$event` | object | All events | Event object |
| `$item` | any | Loop, Table events | Current item |
| `$index` | number | Loop, Table events | Current index |

## Multiple Handlers

Execute multiple actions:

<CodeBlock lang="json">
```json
{
  "events": {
    "on_click": [
      { "type": "set_loading", "key": "submit", "value": true },
      { "type": "call_api", "api": "submit" },
      { "type": "set_loading", "key": "submit", "value": false },
      { "type": "show_toast", "message": "Done!" }
    ]
  }
}
```
</CodeBlock>

## Conditional Event Handling

<CodeBlock lang="json">
```json
{
  "events": {
    "on_click": [
      {
        "type": "conditional",
        "condition": "{{state.canSubmit}}",
        "then": [
          { "type": "call_api", "api": "submit" }
        ],
        "else": [
          { "type": "show_toast", "message": "Cannot submit", "level": "warning" }
        ]
      }
    ]
  }
}
```
</CodeBlock>

## Best Practices

1. **Keep handlers focused** - One purpose per handler
2. **Use loading states** - Show progress for async operations
3. **Handle errors** - Provide user feedback
4. **Debounce expensive operations** - Use `debouncedAction`
5. **Clean up on unmount** - Prevent memory leaks
