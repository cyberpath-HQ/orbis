## Expressions

Expressions in Orbis use double curly braces `{{expression}}` for dynamic value evaluation.

## Basic Syntax

### Interpolation

<CodeBlock lang="json">
```json
"Hello, {{state.name}}!"
"Count: {{state.count}}"
```
</CodeBlock>

### In Properties

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "text": "{{state.message}}",
  "visible": "{{state.showMessage}}",
  "className": "{{state.isError ? 'text-red-500' : 'text-green-500'}}"
}
```
</CodeBlock>

## Expression Types

### String Expressions

<CodeBlock lang="json">
```json
"{{state.firstName}} {{state.lastName}}"
"Hello, {{state.name}}!"
"Total: ${{state.total.toFixed(2)}}"
```
</CodeBlock>

### Numeric Expressions

<CodeBlock lang="json">
```json
"{{state.count + 1}}"
"{{state.price * state.quantity}}"
"{{Math.round(state.average * 100) / 100}}"
```
</CodeBlock>

### Boolean Expressions

<CodeBlock lang="json">
```json
"{{state.isActive}}"
"{{state.count > 0}}"
"{{state.user && state.user.isAdmin}}"
```
</CodeBlock>

### Ternary Expressions

<CodeBlock lang="json">
```json
"{{state.count > 0 ? 'Has items' : 'Empty'}}"
"{{state.loading ? 'Loading...' : 'Ready'}}"
"{{state.error ? state.error.message : 'Success'}}"
```
</CodeBlock>

## Operators

### Comparison Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `===` | Strict equal | `{{state.status === 'active'}}` |
| `!==` | Not equal | `{{state.count !== 0}}` |
| `>` | Greater than | `{{state.age > 18}}` |
| `<` | Less than | `{{state.price < 100}}` |
| `>=` | Greater or equal | `{{state.score >= 60}}` |
| `<=` | Less or equal | `{{state.quantity <= 10}}` |

### Logical Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `&&` | Logical AND | `{{state.a && state.b}}` |
| `\|\|` | Logical OR | `{{state.a \|\| state.b}}` |
| `!` | Logical NOT | `{{!state.loading}}` |

### Arithmetic Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `+` | Addition | `{{state.a + state.b}}` |
| `-` | Subtraction | `{{state.a - state.b}}` |
| `*` | Multiplication | `{{state.price * state.qty}}` |
| `/` | Division | `{{state.total / state.count}}` |
| `%` | Modulo | `{{state.index % 2}}` |

### String Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `+` | Concatenation | `{{state.first + ' ' + state.last}}` |

## Context Variables

### State Context

<CodeBlock lang="json">
```json
"{{state.fieldName}}"
"{{state.nested.field}}"
"{{state.array[0]}}"
```
</CodeBlock>

### Form Context

<CodeBlock lang="json">
```json
"{{form.formId}}"
"{{form.formId.fieldName}}"
"{{form.formId.$valid}}"
"{{form.formId.$dirty}}"
"{{form.formId.$errors}}"
"{{form.formId.$errors.fieldName}}"
```
</CodeBlock>

### Event Context

<CodeBlock lang="json">
```json
"{{$value}}"     // Current input value
"{{$event}}"     // Full event object
"{{$index}}"     // Loop iteration index
"{{$item}}"      // Current loop item
```
</CodeBlock>

### Response Context

<CodeBlock lang="json">
```json
"{{$response}}"           // Full API response
"{{$response.data}}"      // Response data
"{{$response.items}}"     // Response items
"{{$error}}"              // Error object
"{{$error.message}}"      // Error message
```
</CodeBlock>

### Navigation Context

<CodeBlock lang="json">
```json
"{{params.id}}"           // Route parameters
"{{query.search}}"        // Query parameters
"{{route.path}}"          // Current route path
```
</CodeBlock>

### Loading Context

<CodeBlock lang="json">
```json
"{{loading.keyName}}"     // Loading state
"{{loading.submit}}"      // Submit loading
"{{loading.fetch}}"       // Fetch loading
```
</CodeBlock>

### Special Values

<CodeBlock lang="json">
```json
"{{$now}}"                // Current timestamp
"{{$window}}"             // Window object
"{{$window.innerWidth}}"  // Window width
```
</CodeBlock>

## Advanced Patterns

### Nullish Coalescing

<CodeBlock lang="json">
```json
"{{state.user?.name ?? 'Anonymous'}}"
```
</CodeBlock>

### Optional Chaining

<CodeBlock lang="json">
```json
"{{state.data?.items?.[0]?.name}}"
```
</CodeBlock>

### Template Literals

<CodeBlock lang="json">
```json
"{{`Hello, ${state.name}!`}}"
```
</CodeBlock>

### Spread Operator

<CodeBlock lang="json">
```json
"{{ { ...state.user, name: 'Updated' } }}"
"{{ [...state.items, newItem] }}"
```
</CodeBlock>

### Destructuring in Callbacks

<CodeBlock lang="json">
```json
"{{state.items.map(({ id, name }) => `${id}: ${name}`).join(', ')}}"
```
</CodeBlock>

## Expression Contexts

### Component Properties

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "text": "{{state.message}}",
  "visible": "{{state.showMessage}}",
  "className": "{{state.isActive ? 'active' : 'inactive'}}"
}
```
</CodeBlock>

### Action Parameters

<CodeBlock lang="json">
```json
{
  "type": "call_api",
  "api": "getUser",
  "params": {
    "id": "{{params.userId}}",
    "includeDetails": "{{state.showDetails}}"
  }
}
```
</CodeBlock>

### Conditional Rendering

<CodeBlock lang="json">
```json
{
  "type": "Conditional",
  "conditions": [
    {
      "when": "{{state.status === 'loading'}}",
      "render": { "type": "Skeleton" }
    },
    {
      "when": "{{state.status === 'error'}}",
      "render": { "type": "Alert", "variant": "destructive" }
    }
  ],
  "fallback": { "type": "Text", "text": "Ready" }
}
```
</CodeBlock>

### Loop Rendering

<CodeBlock lang="json">
```json
{
  "type": "Loop",
  "items": "{{state.items}}",
  "itemAs": "item",
  "indexAs": "index",
  "render": {
    "type": "Text",
    "text": "{{$index + 1}}. {{$item.name}}"
  }
}
```
</CodeBlock>

## Best Practices

1. **Keep expressions simple** - Extract complex logic to state
2. **Use safe access** - Handle null/undefined values
3. **Avoid side effects** - Expressions should be pure
4. **Use meaningful variable names** - Self-documenting code
5. **Test edge cases** - Empty arrays, null values, etc.
