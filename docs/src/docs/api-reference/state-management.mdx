## State Management

State management in Orbis plugins is declarative and schema-driven. This reference covers all state-related APIs.

## State Initialization

Define initial state in your page definition:

<CodeBlock lang="json">
```json
{
  "id": "myPage",
  "title": "My Page",
  "initialState": {
    "count": 0,
    "user": null,
    "items": [],
    "settings": {
      "theme": "light",
      "notifications": true
    }
  }
}
```
</CodeBlock>

## State Types

### Primitive Types

| Type | Example | Access Pattern |
|------|---------|----------------|
| `string` | `"hello"` | `{{state.message}}` |
| `number` | `42` | `{{state.count}}` |
| `boolean` | `true` | `{{state.isActive}}` |
| `null` | `null` | `{{state.user}}` |

### Complex Types

| Type | Example | Access Pattern |
|------|---------|----------------|
| `object` | `{"name": "John"}` | `{{state.user.name}}` |
| `array` | `[1, 2, 3]` | `{{state.items[0]}}` |
| `nested` | `{"a": {"b": 1}}` | `{{state.a.b}}` |

## State Access

### Basic Access

<CodeBlock lang="json">
```json
"{{state.fieldName}}"
```
</CodeBlock>

### Nested Access

<CodeBlock lang="json">
```json
"{{state.user.profile.name}}"
"{{state.settings.display.theme}}"
```
</CodeBlock>

### Array Access

<CodeBlock lang="json">
```json
"{{state.items[0]}}"
"{{state.items[state.selectedIndex]}}"
"{{state.items.length}}"
```
</CodeBlock>

### Safe Access

Use optional chaining for potentially undefined values:

<CodeBlock lang="json">
```json
"{{state.user?.name}}"
"{{state.items?.[0]?.title}}"
```
</CodeBlock>

## State Mutations

### update_state Action

<CodeBlock lang="json">
```json
{
  "type": "update_state",
  "path": "fieldName",
  "value": "newValue"
}
```
</CodeBlock>

### Path Syntax

| Path | Target | Example |
|------|--------|---------|
| `field` | Root field | `count` |
| `a.b` | Nested field | `user.name` |
| `arr[0]` | Array index | `items[0]` |
| `arr[]` | Array push | `items[]` |
| `a.b.c` | Deep nested | `settings.ui.theme` |

### Value Types

**Static values:**

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "count", "value": 10 }
{ "type": "update_state", "path": "name", "value": "John" }
{ "type": "update_state", "path": "active", "value": true }
```
</CodeBlock>

**Expression values:**

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "count", "value": "{{state.count + 1}}" }
{ "type": "update_state", "path": "fullName", "value": "{{state.firstName}} {{state.lastName}}" }
```
</CodeBlock>

**Object values:**

<CodeBlock lang="json">
```json
{
  "type": "update_state",
  "path": "user",
  "value": {
    "name": "{{$response.name}}",
    "email": "{{$response.email}}"
  }
}
```
</CodeBlock>

**Array values:**

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "items", "value": "{{$response.data}}" }
{ "type": "update_state", "path": "selectedIds", "value": [] }
```
</CodeBlock>

## Array Operations

### Push to Array

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "items[]", "value": "{{newItem}}" }
```
</CodeBlock>

### Update Array Item

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "items[{{state.selectedIndex}}]", "value": "{{updatedItem}}" }
```
</CodeBlock>

### Remove from Array

<CodeBlock lang="json">
```json
{
  "type": "update_state",
  "path": "items",
  "value": "{{state.items.filter((_, i) => i !== state.selectedIndex)}}"
}
```
</CodeBlock>

### Clear Array

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "items", "value": [] }
```
</CodeBlock>

### Map Array

<CodeBlock lang="json">
```json
{
  "type": "update_state",
  "path": "items",
  "value": "{{state.items.map(item => ({ ...item, selected: false }))}}"
}
```
</CodeBlock>

## Object Operations

### Update Object Property

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "user.name", "value": "Jane" }
```
</CodeBlock>

### Merge Objects

<CodeBlock lang="json">
```json
{
  "type": "update_state",
  "path": "settings",
  "value": "{{ { ...state.settings, theme: 'dark' } }}"
}
```
</CodeBlock>

### Clear Object

<CodeBlock lang="json">
```json
{ "type": "update_state", "path": "user", "value": null }
```
</CodeBlock>

## Computed State

Use expressions for computed values:

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "text": "Total: {{state.items.reduce((sum, item) => sum + item.price, 0)}}"
}
```
</CodeBlock>

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "text": "{{state.items.filter(i => i.active).length}} active items"
}
```
</CodeBlock>

## State in Components

### Conditional Visibility

<CodeBlock lang="json">
```json
{
  "type": "Alert",
  "visible": "{{state.showWarning}}",
  "message": "Warning message"
}
```
</CodeBlock>

### Dynamic Content

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "text": "Welcome, {{state.user.name}}!"
}
```
</CodeBlock>

### Disabled State

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "disabled": "{{state.isSubmitting || !state.isValid}}"
}
```
</CodeBlock>

### Loading State

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "loading": "{{loading.submit}}",
  "text": "{{loading.submit ? 'Saving...' : 'Save'}}"
}
```
</CodeBlock>

## State Persistence

State can be persisted across sessions:

<CodeBlock lang="json">
```json
{
  "initialState": {
    "theme": "light"
  },
  "stateConfig": {
    "persist": ["theme", "sidebarOpen"]
  }
}
```
</CodeBlock>

## State Reset

Reset state to initial values:

<CodeBlock lang="json">
```json
{
  "type": "update_state",
  "path": "user",
  "value": null
}
```
</CodeBlock>

Reset multiple fields:

<CodeBlock lang="json">
```json
{
  "type": "sequence",
  "actions": [
    { "type": "update_state", "path": "user", "value": null },
    { "type": "update_state", "path": "items", "value": [] },
    { "type": "update_state", "path": "loading", "value": false }
  ]
}
```
</CodeBlock>

## Best Practices

1. **Initialize all state** - Define all fields in `initialState`
2. **Use appropriate types** - Match state types to usage
3. **Keep state flat** - Avoid deeply nested structures when possible
4. **Use meaningful names** - Self-documenting field names
5. **Handle null/undefined** - Use safe access patterns
6. **Minimize state** - Only store what's needed
