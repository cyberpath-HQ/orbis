# Expressions

Expressions are the core mechanism for making Orbis UIs dynamic. They allow you to compute values, evaluate conditions, and reference data at runtime.

## Syntax

Expressions use double curly braces:

<CodeBlock lang="text">
```text
{{expression}}
```
</CodeBlock>

They can appear in any string property:

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "content": "Hello, {{state.username}}!"
}
```
</CodeBlock>

## Expression Context

Every expression has access to various data sources:

### State

Page state values:

<CodeBlock lang="json">
```json
"{{state.count}}"
"{{state.user.name}}"
"{{state.items[0]}}"
```
</CodeBlock>

### Context

Contextual data (from loops, modals, etc.):

<CodeBlock lang="json">
```json
"{{context.modalData}}"
"{{context.parentId}}"
```
</CodeBlock>

### Special Variables

Event and iteration context:

| Variable | Description | Available In |
|----------|-------------|--------------|
| `$event` | Event object | Event handlers |
| `$event.value` | Input value | on_change handlers |
| `$event.target` | Event target | All event handlers |
| `$item` | Current item | Loop, List, Table |
| `$index` | Current index | Loop, List, Table |
| `$row` | Current row | Table components |
| `$response` | API response | call_api on_success |
| `$response.data` | Response body | call_api on_success |
| `$error` | Error object | call_api on_error |

## Data Access

### Simple Properties

<CodeBlock lang="json">
```json
"{{state.username}}"
```
</CodeBlock>

### Nested Properties

Use dot notation:

<CodeBlock lang="json">
```json
"{{state.user.profile.avatar}}"
```
</CodeBlock>

### Array Elements

Use bracket notation:

<CodeBlock lang="json">
```json
"{{state.items[0]}}"
"{{state.items[state.selectedIndex]}}"
```
</CodeBlock>

### Object Keys

<CodeBlock lang="json">
```json
"{{state.users[userId]}}"
```
</CodeBlock>

## Operators

### Arithmetic

<CodeBlock lang="json">
```json
"{{state.count + 1}}"
"{{state.price * state.quantity}}"
"{{state.total - state.discount}}"
"{{state.amount / 100}}"
"{{state.value % 2}}"
```
</CodeBlock>

### Comparison

<CodeBlock lang="json">
```json
"{{state.count > 0}}"
"{{state.count >= 10}}"
"{{state.age < 18}}"
"{{state.age <= 65}}"
"{{state.status == 'active'}}"
"{{state.status === 'active'}}"
"{{state.type != 'deleted'}}"
"{{state.type !== 'deleted'}}"
```
</CodeBlock>

### Logical

<CodeBlock lang="json">
```json
"{{state.isActive && state.isVerified}}"
"{{state.isAdmin || state.isModerator}}"
"{{!state.isLoading}}"
```
</CodeBlock>

### Ternary

<CodeBlock lang="json">
```json
"{{state.count > 0 ? 'Has items' : 'Empty'}}"
"{{state.isActive ? 'active' : 'inactive'}}"
```
</CodeBlock>

## String Operations

### Concatenation

<CodeBlock lang="json">
```json
"{{state.firstName + ' ' + state.lastName}}"
"Hello, {{state.name}}!"
```
</CodeBlock>

### Template Strings

Mix static and dynamic content:

<CodeBlock lang="json">
```json
"User {{state.username}} has {{state.points}} points"
```
</CodeBlock>

## Common Patterns

### Conditional Visibility

<CodeBlock lang="json">
```json
{
  "type": "Alert",
  "visible": "{{state.errors.length > 0}}",
  "message": "There are errors"
}
```
</CodeBlock>

### Dynamic Classes

<CodeBlock lang="json">
```json
{
  "type": "Container",
  "className": "{{state.isActive ? 'bg-primary' : 'bg-muted'}}"
}
```
</CodeBlock>

### Computed Labels

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "label": "{{state.isEditing ? 'Save' : 'Edit'}}"
}
```
</CodeBlock>

### Disabled States

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "disabled": "{{state.isLoading || !state.isValid}}"
}
```
</CodeBlock>

### Loop Item Access

<CodeBlock lang="json">
```json
{
  "type": "Loop",
  "dataSource": "state:items",
  "template": {
    "type": "Text",
    "content": "{{$index + 1}}. {{$item.name}}"
  }
}
```
</CodeBlock>

### Event Values

<CodeBlock lang="json">
```json
{
  "type": "Field",
  "events": {
    "on_change": [
      {
        "type": "update_state",
        "path": "searchQuery",
        "value": "$event.value"
      }
    ]
  }
}
```
</CodeBlock>

### API Response Handling

<CodeBlock lang="json">
```json
{
  "type": "call_api",
  "api": "my-plugin.getData",
  "on_success": [
    {
      "type": "update_state",
      "path": "data",
      "value": "$response.data"
    }
  ],
  "on_error": [
    {
      "type": "show_toast",
      "message": "Error: {{$error.message}}",
      "level": "error"
    }
  ]
}
```
</CodeBlock>

## Boolean Expressions

Boolean expressions are evaluated for properties like `visible`, `disabled`, `loading`:

<CodeBlock lang="json">
```json
{
  "visible": true,              // Static true
  "visible": false,             // Static false
  "visible": "{{state.show}}",  // Dynamic boolean
  "visible": "{{state.count > 0 && state.isActive}}"  // Complex condition
}
```
</CodeBlock>

### Truthy/Falsy Evaluation

Expressions follow JavaScript truthy/falsy rules:

| Value | Boolean Result |
|-------|----------------|
| `true` | `true` |
| `false` | `false` |
| `0` | `false` |
| `""` | `false` |
| `null` | `false` |
| `undefined` | `false` |
| `[]` | `true` (empty array) |
| `{}` | `true` (empty object) |
| Any non-zero number | `true` |
| Any non-empty string | `true` |

## Expression Caching

Orbis caches expression results for performance:

- Parsed expressions are memoized
- Unchanged state produces cached results
- Cache is invalidated on state changes

This means repeated re-renders don't re-parse expressions.

## Error Handling

Invalid expressions degrade gracefully:

<CodeBlock lang="json">
```json
// Missing property - returns undefined/empty
"{{state.nonexistent}}"  // → ""

// Type error - returns empty
"{{state.number.toUpperCase()}}"  // → "" (logged in development)
```
</CodeBlock>

In development mode, expression errors are logged to help debugging.

## Limitations

### No Function Calls

Expressions cannot call arbitrary functions:

<CodeBlock lang="json">
```json
// ❌ Not supported
"{{state.items.map(i => i.name)}}"
"{{Date.now()}}"
"{{Math.random()}}"
```
</CodeBlock>

Use computed state or actions instead.

### No Assignments

Expressions are read-only:

<CodeBlock lang="json">
```json
// ❌ Not supported
"{{state.count = 5}}"
"{{state.count++}}"
```
</CodeBlock>

Use the `update_state` action to modify state.

### No Multi-Statement

Expressions are single expressions:

<CodeBlock lang="json">
```json
// ❌ Not supported
"{{let x = 1; x + 2}}"
```
</CodeBlock>

## Best Practices

### Keep Expressions Simple

<CodeBlock lang="json">
```json
// Good
"{{state.isLoggedIn && state.hasPermission}}"

// ❌ Complex - hard to debug
"{{state.users.filter(u => u.active).length > 0 && state.settings.feature}}"
```
</CodeBlock>

### Use Descriptive State Names

<CodeBlock lang="json">
```json
// Good
"{{state.isSubmitButtonDisabled}}"

// ❌ Unclear
"{{state.d}}"
```
</CodeBlock>

### Prefer Pre-Computed State

For complex logic, compute in actions:

<CodeBlock lang="json">
```json
// Action that computes value
{
  "type": "update_state",
  "path": "canSubmit",
  "value": "{{state.isValid && !state.isLoading && state.hasChanges}}"
}

// Then use simply
{
  "disabled": "{{!state.canSubmit}}"
}
```
</CodeBlock>

### Handle Missing Data

<CodeBlock lang="json">
```json
// Handle potential undefined
"{{state.user?.name || 'Anonymous'}}"

// Or use conditional rendering
{
  "visible": "{{state.user}}",
  "content": "{{state.user.name}}"
}
```
</CodeBlock>

## Debugging Expressions

### Development Mode

Enable debug logging:

<CodeBlock lang="bash">
```bash
RUST_LOG=debug bun run tauri dev
```
</CodeBlock>

### Test Expressions

Use the console to test:

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "content": "DEBUG: {{state.myValue}}"
}
```
</CodeBlock>

### Check Expression Results

Add debug output:

<CodeBlock lang="json">
```json
{
  "type": "Conditional",
  "condition": "{{state.shouldShow}}",
  "then": {
    "type": "Text",
    "content": "Condition is true"
  },
  "else": {
    "type": "Text", 
    "content": "Condition is false: {{state.shouldShow}}"
  }
}
```
</CodeBlock>
