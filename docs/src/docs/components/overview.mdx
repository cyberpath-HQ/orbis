Orbis provides 35+ UI components organized into categories. Each component is defined by a JSON schema and rendered by the SchemaRenderer.

## Common Properties

All components share these base properties:

<CodeBlock lang="typescript">
```typescript
interface BaseSchema {
  type: string;           // Component type (required)
  visible?: string;       // Visibility expression
  className?: string;     // CSS classes (Tailwind)
  style?: object;         // Inline styles
  ariaLabel?: string;     // Accessibility label
}
```
</CodeBlock>

### type

The component type identifier (required).

<CodeBlock lang="json">
```json
{ "type": "Button" }
```
</CodeBlock>

### visible

Expression controlling visibility:

<CodeBlock lang="json">
```json
{
  "type": "Card",
  "visible": "{{state.showCard}}"
}
```
</CodeBlock>

Supports complex expressions:
<CodeBlock lang="json">
```json
{ "visible": "{{state.count}} > 0 && {{state.isActive}}" }
```
</CodeBlock>

### className

Tailwind CSS classes:

<CodeBlock lang="json">
```json
{
  "type": "Container",
  "className": "p-6 bg-white rounded-lg shadow"
}
```
</CodeBlock>

## Event Handling

Components with interactivity have events:

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "label": "Click Me",
  "events": {
    "on_click": [
      { "type": "update_state", "path": "count", "value": "{{state.count}} + 1" }
    ]
  }
}
```
</CodeBlock>

Available events vary by component. See individual component docs.

## Expressions in Properties

Most string properties support expressions:

<CodeBlock lang="json">
```json
{
  "type": "Text",
  "content": "Hello, {{state.userName}}!"
}
```
</CodeBlock>

See [Expressions](../core-concepts/expressions) for full syntax.

## Usage Patterns

### Container with Children

<CodeBlock lang="json">
```json
{
  "type": "Container",
  "className": "p-6",
  "children": [
    { "type": "Heading", "text": "Title", "level": 1 },
    { "type": "Text", "content": "Description text here." }
  ]
}
```
</CodeBlock>

### Data-Bound Component

<CodeBlock lang="json">
```json
{
  "type": "Table",
  "dataSource": "state:items",
  "columns": [
    { "key": "name", "label": "Name" },
    { "key": "status", "label": "Status" }
  ]
}
```
</CodeBlock>

### Interactive Component

<CodeBlock lang="json">
```json
{
  "type": "Button",
  "label": "Save",
  "variant": "default",
  "events": {
    "on_click": [
      { "type": "call_api", "api": "saveData" },
      { "type": "show_toast", "message": "Saved!" }
    ]
  }
}
```
</CodeBlock>

### Conditional Rendering

<CodeBlock lang="json">
```json
{
  "type": "Conditional",
  "condition": "{{state.isLoading}}",
  "then": { "type": "Skeleton" },
  "else": { "type": "Card", "content": { "..." } }
}
```
</CodeBlock>

## Styling

### Tailwind Classes

All components support `className` for Tailwind:

<CodeBlock lang="json">
```json
{
  "type": "Card",
  "className": "bg-gradient-to-r from-blue-500 to-purple-500 text-white"
}
```
</CodeBlock>

### Responsive Classes

<CodeBlock lang="json">
```json
{
  "type": "Grid",
  "columns": { "sm": 1, "md": 2, "lg": 3 },
  "className": "gap-4 md:gap-6"
}
```
</CodeBlock>

### Dark Mode

Use dark: prefix:

<CodeBlock lang="json">
```json
{
  "className": "bg-white dark:bg-gray-900 text-gray-900 dark:text-white"
}
```
</CodeBlock>
